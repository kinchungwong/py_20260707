## What got built

- **`src/pypiano_2607/app.py`** — `PianoApp`, the playable shell, promoted from the
  frozen `playable_instrument` spike's procedural `state` dict into a class. Wires
  PyGame loop → `InputRouter` → `EventQueue` → `PolySynth` →
  `sounddevice.OutputStream(latency='low')`. Same `dispatch(event)` runs live and under
  the tests; `emit()`, `all_notes_off()`, `run()`, `close()`. **pygame AND sounddevice
  import lazily** (inside methods), so the module imports headless.
- **`src/pypiano_2607/mouse.py`** — `MouseGlissando`, the single-mouse-note drag state
  machine deferred out of the input layer, now a **pygame-free, events-free** library

- **`src/pypiano_2607/gui/__init__.py`** — window geometry `MARGIN=24`,
  `WIN_W`/`WIN_H` (derived from the keyboard `WIDTH`/`HEIGHT`), promoted from the spike.
- **Public-API promotion** (deferred from increment 2): `InputRouter`, `MouseGlissando`,
  `PianoApp` now exported from `pypiano_2607/__init__.py`.
- **`examples/play_app.py`** — the runnable CLI (`--voice piano|sine`, headless
  `--selftest`), mirroring `play_synth.py`; keeps argparse/`__main__`/device out of the

- **Tests:** `test_app.py` (headless — synthetic PyGame events through the live

## Decisions & why (canonical facts in `../plans/` + `../memory/`)

- **Faithful port** (human decision): reproduce the spike's behavior in a class, **no
  new features** — no velocity, sustain, or octave-shift. The upper octave (MIDI 73–83)
  stays mouse-only, as in the spike.
- **Promote glissando + geometry to the library** (human decision) rather than keeping
  them app-local; hence `mouse.py` + the `gui` geometry constants, both re-exported.
- **Altitude:** the reusable `PianoApp` class lives in `src/`; the CLI lives in
  `examples/play_app.py` — every other `src/` module is import-only.
- **The spike→library reconciliations applied:** 5-field `NoteEvent(kind, sounder_id,
  freq, source, t)` (the spike's 4-field construct in `_all_notes_off` was wrong arity →
  now `freq=None`); `ev.midi`→`ev.sounder_id`; `synth.voice_midi`→`synth.voice_sounder`;
  the router already mints `freq`, so the press/release/emit glue is otherwise unchanged.
- **Additive, frozen-increment discipline held:** only the two package `__init__` files +
  `test_gui_package.py` were edited (all additive); increment-1/2 **logic** files and all
  spikes are byte-identical (`git diff claude/spikes` empty; spike net 14/14).

## How it was built

Plan-first (human-approved, committed `b5384fc`), then implemented coherently by one
author (the modules are tightly coupled — fan-out would risk seam mismatches), then a
**build → test → adversarial-verify** pass. A dimension-split review workflow (3
reviewers → per-finding adversarial verify) confirmed correctness + invariants clean and
surfaced **one real gap**: no test guarded the glissando reset in `all_notes_off()` while
a mouse drag was active (a removed reset would silently reintroduce a phantom-note bug).
Closed with `test_focus_loss_resets_an_active_mouse_drag`, **mutation-tested** (the guard
fails as designed when the reset is deleted).
